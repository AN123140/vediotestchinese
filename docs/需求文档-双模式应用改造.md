# 视频字幕生成软件 - 双模式应用改造需求文档

## 1. 项目概述

### 1.1 改造目标
将现有的纯桌面应用改造为**双模式应用**，同时支持桌面端（Electron）和网页端（浏览器），最大化代码复用率，兼顾功能完整性和便携性。

### 1.2 版本信息
- **版本号**：v3.2.0
- **开发周期**：预计14天
- **基于版本**：v3.1.0（桌面版）

### 1.3 双模式定位

| 维度 | 桌面端（Electron） | 网页端（Browser） |
|------|-------------------|------------------|
| **目标场景** | 专业用户、批量处理 | 临时使用、跨设备访问 |
| **核心优势** | 功能完整、离线运行、大文件支持 | 无需安装、即开即用、跨平台 |
| **功能范围** | 100%功能 | 80%核心功能 |
| **性能表现** | 本地模型、GPU加速 | 云端API/WebAssembly |

## 2. 功能需求

### 2.1 双模式架构

#### 2.1.1 环境检测机制
- **检测方式**：检查 `window.electronAPI` 是否存在
- **环境标识**：
  - 桌面端：`environment = 'electron'`
  - 网页端：`environment = 'browser'`
- **动态适配**：运行时自动检测，无需编译时区分

#### 2.1.2 功能矩阵

| 功能模块 | 桌面端 | 网页端 | 网页端降级方案 |
|---------|--------|--------|---------------|
| 视频导入 | ✅ 原生对话框+拖拽 | ✅ `<input>`+拖拽 | 无 |
| 视频播放 | ✅ 本地文件路径 | ✅ Blob URL | 自动转换 |
| 语音识别 | ✅ 本地Whisper | ⚠️ 云端API/WASM | 提示配置API Key |
| 字幕编辑 | ✅ 完整功能 | ✅ 完整功能 | 无 |
| 字幕导出 | ✅ 本地文件系统 | ✅ 浏览器下载 | 自动降级 |
| 字幕压制 | ✅ FFmpeg本地 | ❌ 不支持 | 提示使用桌面版 |
| 批量处理 | ✅ 完整队列 | ⚠️ 单文件处理 | 限制并发数 |
| 系统托盘 | ✅ 支持 | ❌ 不支持 | 隐藏相关UI |
| 全局快捷键 | ✅ 支持 | ❌ 不支持 | 隐藏相关设置 |
| 开机启动 | ✅ 支持 | ❌ 不支持 | 隐藏相关设置 |
| 本地存储 | ✅ 文件系统 | ✅ IndexedDB | 自动适配 |

### 2.2 视频处理适配

#### 2.2.1 文件导入适配
**桌面端**：
- 使用 `dialog.showOpenDialog` 原生文件对话框
- 返回文件路径和元数据
- 支持多文件选择

**网页端**：
- 使用 `<input type="file" accept="video/*">`
- 返回 File 对象
- 使用 `URL.createObjectURL()` 生成播放URL

**统一接口**：
```javascript
// adapters/fileAdapter.js
export async function openVideoFile() {
  if (isElectron()) {
    return await window.electronAPI.openVideoDialog()
  } else {
    return await openBrowserFileDialog()
  }
}
```

#### 2.2.2 视频播放适配
**桌面端**：
- 直接使用 `file:///` 协议路径
- 支持大文件（>4GB）
- 无内存限制

**网页端**：
- 使用 `URL.createObjectURL(file)` 生成 Blob URL
- 文件大小受限于浏览器内存（通常<2GB）
- 需要及时 `URL.revokeObjectURL()` 释放内存

### 2.3 语音识别适配

#### 2.3.1 识别策略选择

| 方式 | 桌面端 | 网页端 | 准确率 | 成本 | 隐私 |
|------|--------|--------|--------|------|------|
| 本地Whisper | ✅ 首选 | ❌ 不支持 | ⭐⭐⭐⭐⭐ | 免费 | 高 |
| WebAssembly tiny | ✅ 备选 | ✅ 首选 | ⭐⭐⭐ | 免费 | 高 |
| 云端API | ✅ 备选 | ✅ 备选 | ⭐⭐⭐⭐ | 付费 | 低 |
| Web Speech API | ❌ 不支持 | ✅ 备选 | ⭐⭐ | 免费 | 中 |

#### 2.3.2 默认策略
- **桌面端**：优先使用本地Whisper模型，可选云端API
- **网页端**：
  1. 优先：WebAssembly tiny模型（离线、免费）
  2. 备选：用户配置的云端API（百度/讯飞/阿里云）
  3. 备选：浏览器Web Speech API（仅限Chrome）

#### 2.3.3 云端API配置
支持以下服务商：
- 百度智能云语音识别
- 讯飞开放平台
- 阿里云智能语音
- Azure Speech Services
- Google Cloud Speech-to-Text

配置方式：
- 用户在设置中填写 API Key / Secret
- 支持多服务商切换
- 本地加密存储密钥

### 2.4 存储适配

#### 2.4.1 配置存储
**桌面端**：
- 使用本地JSON文件存储
- 路径：`%APPDATA%/subtitle-app/config.json`

**网页端**：
- 使用 LocalStorage / IndexedDB
- 键值对存储

**统一接口**：
```javascript
// adapters/storageAdapter.js
export const storage = {
  get: async (key) => { /* 自动适配 */ },
  set: async (key, value) => { /* 自动适配 */ },
  remove: async (key) => { /* 自动适配 */ }
}
```

#### 2.4.2 字幕数据存储
**桌面端**：
- 自动保存到本地文件
- 支持项目文件格式（.subtitleproj）

**网页端**：
- 使用 IndexedDB 存储
- 支持导出/导入项目文件

#### 3.1.2 适配器层设计

**fileAdapter.js**
```javascript
import { isElectron } from '../utils/environment.js'

export const fileAdapter = {
  // 打开视频文件
  async openVideo() {
    if (isElectron()) {
      return await window.electronAPI.openVideoDialog()
    } else {
      return await openBrowserFileDialog()
    }
  },

  // 获取视频播放URL
  async getVideoUrl(file) {
    if (isElectron()) {
      return 'file:///' + file.path.replace(/\\/g, '/')
    } else {
      return URL.createObjectURL(file)
    }
  },

  // 释放视频URL（网页端需要）
  revokeVideoUrl(url) {
    if (!isElectron() && url.startsWith('blob:')) {
      URL.revokeObjectURL(url)
    }
  },

  // 保存字幕文件
  async saveSrt(content, defaultName = 'subtitles.srt') {
    if (isElectron()) {
      return await window.electronAPI.saveSrtDialog(content)
    } else {
      return downloadInBrowser(content, defaultName)
    }
  }
}
```

**recognizeAdapter.js**
```javascript
import { isElectron } from '../utils/environment.js'

export const recognizeAdapter = {
  async recognize(videoSource, options = {}) {
    const strategy = this.selectStrategy(options)
    
    switch (strategy) {
      case 'local':
        return await recognizeLocal(videoSource, options)
      case 'wasm':
        return await recognizeWASM(videoSource, options)
      case 'cloud':
        return await recognizeCloud(videoSource, options)
      case 'web-speech':
        return await recognizeWebSpeech(videoSource, options)
      default:
        throw new Error('无可用的识别方式')
    }
  },

  selectStrategy(options) {
    if (isElectron() && options.useLocal !== false) {
      return 'local'  // 桌面端优先本地
    }
    if (options.apiKey) {
      return 'cloud'  // 有API Key用云端
    }
    if (isWasmSupported()) {
      return 'wasm'   // 支持WASM用本地tiny模型
    }
    if (isWebSpeechSupported()) {
      return 'web-speech'  // 最后尝试Web Speech
    }
    return null
  }
}
```

**storageAdapter.js**
```javascript
import { isElectron } from '../utils/environment.js'

export const storageAdapter = {
  async get(key) {
    if (isElectron()) {
      return await window.electronAPI.getConfig(key)
    } else {
      return localStorage.getItem(key)
    }
  },

  async set(key, value) {
    if (isElectron()) {
      return await window.electronAPI.setConfig(key, value)
    } else {
      localStorage.setItem(key, value)
    }
  },

  async remove(key) {
    if (isElectron()) {
      return await window.electronAPI.removeConfig(key)
    } else {
      localStorage.removeItem(key)
    }
  }
}
```

### 3.2 环境检测工具

**utils/environment.js**
```javascript
// 检测是否在Electron环境
export function isElectron() {
  return typeof window !== 'undefined' && 
         typeof window.electronAPI !== 'undefined'
}

// 检测是否在浏览器环境
export function isBrowser() {
  return !isElectron()
}

// 获取当前环境
export function getEnvironment() {
  return isElectron() ? 'electron' : 'browser'
}

// 检测WebAssembly支持
export function isWasmSupported() {
  return typeof WebAssembly === 'object' &&
         typeof WebAssembly.instantiate === 'function'
}

// 检测Web Speech API支持
export function isWebSpeechSupported() {
  return 'webkitSpeechRecognition' in window ||
         'SpeechRecognition' in window
}
```

## 6. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| WebAssembly模型体积大 | 网页加载慢 | 使用tiny模型，懒加载，显示加载进度 |
| 云端API费用 | 用户成本增加 | 明确提示，提供WASM免费替代方案 |
| 浏览器兼容性 | 部分浏览器不支持 | 检测并提示用户更换浏览器 |
| 大文件处理 | 浏览器内存溢出 | 限制文件大小，友好提示 |
| 代码复杂度增加 | 维护困难 | 适配器层封装，保持组件纯净 |

### 7.2 WebAssembly支持检测
```javascript
function checkWasmSupport() {
  try {
    if (typeof WebAssembly === 'object'
        && typeof WebAssembly.instantiate === 'function') {
      const module = new WebAssembly.Module(
        Uint8Array.of(0x0, 0x61, 0x73, 0x6d, 0x01, 0x00, 0x00, 0x00)
      )
      if (module instanceof WebAssembly.Module)
        return true
    }
  } catch (e) {}
  return false
}
```
